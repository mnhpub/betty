# design-sync notes for Betty

## Fixes and learnings

- **[GENERAL] Fixed-position/portal components render incomplete in the storybook reference capture.** Carbon's `Header`/`SideNav` (position: fixed) and `OverflowMenu` (portal-rendered dropdown) both show only a partial render on the storybook side of the compare sheet — Header's capture drops the fixed header/nav bar entirely (only the content tile shows), and OverflowMenu's capture shows only the closed trigger button even with `open: true` set on the story. In both cases the **preview** (right panel) renders the complete, correct composition. Graded `match` with a note per the rubric's "reference gated" provision — the preview is the more complete/faithful side, not the storybook capture. Both components already carry `cardMode: "single"` overrides (grid/portal escape), which is a separate, correctly-applied fix for the product-card containment — this note is about the *compare* capture specifically, not the card render.
- **`@carbon/icons-react` bloats the bundle unconditionally.** The converter's `[ICON_PKG]` auto-detection merges the *entire* ~2,740-icon package onto `window.Carbon` because `@carbon/react` itself declares it as a real dependency — this fires regardless of `cfg.extraEntries` contents or what any story imports. Confirmed empirically: removing it from config and switching all of Betty's own icon usage to `lucide-react` did NOT shrink the bundle; icons-react was still auto-included. Icons alone cost ~4.85MB, which combined with `@carbon/charts-react` (~850KB for `LineChart`) exceeded the upload's 5MB-per-file cap. **Resolution**: dropped `LineChart`/`@carbon/charts-react` from the synced scope entirely (final bundle ≈4.97MB — icons alone, ~260KB of headroom). LineChart stays in Betty's real app; it's just not part of the Claude Design sync. If more components are added later, watch this headroom closely — there's no config knob to scope icons-react down to only the icons actually used; the only path to more headroom would be an unsanctioned fork of `package-build.mjs`'s `iconSiblings` detection (not attempted — it's not a listed override point and would silently revert on the next `.ds-sync/` refresh).
- Betty's real app (`packages/web`) switched from `@carbon/icons-react` to `lucide-react` for its 3 icon usages (Add→Plus, Notification→Bell, UserAvatar→CircleUser) in `AppShell.tsx` and the Groups `DataTable` page — a deliberate product decision (Carbon's icon set was flagged as "flakey"), independent of the bundle-size fix above (confirmed the bundle-size problem persists regardless, per the bullet above).
- `@carbon/icons-react`'s `package.json` has no `types`/`typings` field, so the converter's `.d.ts`-based export-gate scan (`exportedNames`) can't resolve its real entry (`lib/index.d.ts`) — it falls back to a nonexistent `<pkgDir>/index.d.ts` and returns zero names. This is why standalone icon stories (Add/Notification/UserAvatar, before the lucide-react swap) got dropped with `[TITLE_UNMAPPED]`. Not fixed — worked around by not syncing icons as standalone components (they aren't really catalog-worthy "components" anyway; they render fine as props/children inside other components' bundles).

## Re-sync risks

- The icon-package bundle-size headroom (~260KB) is thin. Adding any new Carbon component to the synced scope risks tripping `[FILE_OVER_5MB]` again — check bundle size after any scope addition before assuming it'll upload cleanly.
- `cfg.provider: GlobalTheme` is set globally for the `g10` theme (matches the real app's `<GlobalTheme theme="g10">` wrapper in `packages/web/src/app/[locale]/layout.tsx`). If Betty's app ever switches themes, update this config to match or previews will render the wrong theme.
- Only 11 of the ~34 Carbon exports Betty actually imports were synced as first-class components (see the story files in `packages/web/src/design-system-stories/`) — the rest (Grid/Column primitives, Table sub-parts like TableHead/TableRow/TableCell, HeaderNavigation/HeaderMenuItem/SideNavItems/SideNavLink, InlineNotification, Tag, TextInput, PasswordInput, Stack, Tile) either got folded into composed stories (e.g., DataTable's story exercises Table/TableHead/TableRow/TableCell/TableToolbar internally) or are covered as standalone stories. Check `src/design-system-stories/` for the full list before assuming something is missing from the sync.
